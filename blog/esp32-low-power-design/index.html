ESP32 低功耗设计实战：从 80mA 到 8μA 的优化之路

记录一个电池供电项目中 ESP32 功耗优化的完整过程，包含硬件选型、固件策略和实测数据对比。

目录

  1. 项目背景与功耗目标
  2. 硬件选型与电路优化
  3. 固件层面功耗优化
  4. 实测数据对比
  5. 总结与经验

项目背景与功耗目标

去年接到一个农业物联网项目，要求设计一款电池供电的环境监测终端，每隔 15 分钟采集一次温湿度、土壤湿度和光照数据，通过 LoRa 上报。设计要求使用两节 AA 电池供电，续航至少 6 个月。

选型阶段我们锁定了 ESP32，原因有三：价格便宜、生态成熟、内置 WiFi/BLE 方便现场调试。但 ESP32 以功耗高闻名——常规运行模式下电流高达 80mA 左右。如何在保持功能的前提下把平均功耗降到 μA 级别，成了整个项目的关键挑战。

最终我们达到了 活跃模式 35mA，深度睡眠 5μA，15 分钟采集周期下 平均功耗约 38μA，两节 AA 碱性电池（~3000mAh）理论续航约 9 年，实际续航超过 18 个月。下面是完整的优化过程。

硬件选型与电路优化

1. 电源管理芯片选择

ESP32 的工作电压范围是 2.3V~3.6V。直接用电池供电会在放电末期电压过低导致系统不稳定。我们选择了 TPS7A05 超低功耗 LDO，其静态电流仅 1μA，相比普通 LDO 的几十 μA 能节省大量待机功耗。

功耗对比

AMS1117-3.3: 静态电流 ~5mA（不适合电池设备）

XC6206: 静态电流 ~2μA（性价比之选）

TPS7A05: 静态电流 ~1μA（最佳选择）

2. 外设功耗控制

项目中使用 SHT30 温湿度传感器和 BH1750 光照传感器。这两个传感器都支持待机模式，通过 ESP32 的 GPIO 控制其电源 VCC，采样时才供电：

// 传感器电源控制宏
#define SENSOR_POWER_PIN  GPIO_NUM_4

void sensor_power_on() {
    gpio_set_level(SENSOR_POWER_PIN, 1);
    vTaskDelay(pdMS_TO_TICKS(10)); // 等待传感器稳定
}

void sensor_power_off() {
    gpio_set_level(SENSOR_POWER_PIN, 0);
}

3. LoRa 模块选型

LoRa 模块选择 Ra-01SH（基于 SX1262），其待机电流仅 0.6μA，远低于传统 SX1278 的 1.6mA 待机电流。单次发送约 50ms，电流 45mA，在 15 分钟周期下可忽略不计。

固件层面功耗优化

4. 深度睡眠模式配置

ESP32 的深度睡眠模式是降低功耗的核心。进入深度睡眠后，CPU 停止运行，大部分 RAM 掉电，仅 RTC 内存保留。我们的策略是：唤醒 → 采样 → 发送 → 深度睡眠。

void enter_deep_sleep() {
    // 关闭外设电源
    sensor_power_off();

    // 配置 RTC 定时器唤醒, 15 分钟
    esp_sleep_enable_timer_wakeup(15 * 60 * 1000000ULL);

    // 配置 GPIO 唤醒（可选，用于调试）
    esp_sleep_enable_ext1_wakeup(BUTTON_MASK, ESP_EXT1_WAKEUP_ANY_LOW);

    // 进入深度睡眠
    esp_deep_sleep_start();
}

5. RTC 内存保留关键数据

默认深度睡眠会清空 DRAM，但 RTC 快速内存（RTC_SLOW_MEM）会在睡眠期间保持供电。我们将采集到的校准数据和计数器放在 RTC 内存中：

RTC_DATA_ATTR int boot_count = 0;
RTC_DATA_ATTR sensor_calib_t calib_data;

void app_main() {
    boot_count++;
    ESP_LOGI("MAIN", "Boot count: %d", boot_count);

    // 恢复校准数据
    apply_calibration(&calib_data);
    ...
}

6. 快速启动优化

ESP32 默认启动流程会初始化所有子系统，耗时约 200ms。通过跳过不必要的初始化可大幅降低每次唤醒的能耗：

优化数据

优化前: 启动耗时 210ms，总耗能 4.2mJ

优化后: 启动耗时 45ms，总耗能 0.9mJ

7. WiFi/BLE 按需开关

虽然最终使用 LoRa 上报，但开发时发现 ESP32 的 WiFi 射频在空闲时仍会周期性扫描信道。明确禁用射频模块是低功耗的关键：

void init_low_power() {
    // 禁用 WiFi 和 BLE
    esp_wifi_stop();
    esp_bt_controller_disable();

    // 设置 WiFi 为无 RF 模式
    wifi_power_save_t ps = { .mode = WIFI_PS_MAX_MODEM };
    esp_wifi_set_ps(&ps);

    // 禁用以太网 MAC
    esp_eth_stop();
}

8. ADC 采样优化

ESP32 的 ADC2 在 WiFi 启用时不可用。我们使用 ADC1，并将采样宽度从 12bit 降到 10bit 以减少采样时间：

adc1_config_width(ADC_WIDTH_BIT_10);   // 默认 12bit → 10bit
adc1_config_channel_atten(ADC1_CHANNEL_0, ADC_ATTEN_DB_11);

uint32_t read_battery() {
    uint32_t raw = 0;
    for (int i = 0; i < 8; i++) {
        raw += adc1_get_raw(ADC1_CHANNEL_0);
    }
    raw /= 8;  // 均值滤波
    return raw;
}

总结与经验

这次功耗优化的核心思路是：能不用的硬件关掉，能少用的固件精简，能短时间的流程压缩。回到最开始的续航需求，38μA 的平均功耗意味着两节 AA 电池可以支撑 9 年理论上限，即使考虑电池自放电和低温影响，实际续航也轻松超过 18 个月。

几个值得分享的经验：

  • 从硬件入手：再好的软件优化也抵不过一个低功耗 LDO 的选择
  • 外设供电控制：通过 MOSFET 或 GPIO 直接控制传感器电源，比用传感器自带的待机模式功耗更低
  • 深睡唤醒频率：每 15 分钟唤醒一次，相比每分钟唤醒，功耗降低约 15 倍
  • 实测曲线很重要：ESP32 的深睡电流在不同批次间有差异，最好在原型阶段用万用表实测
